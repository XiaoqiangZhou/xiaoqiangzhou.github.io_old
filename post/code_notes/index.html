提升效率的命令/技巧

alias命令 在~/.bashrc文件中添加如下命令：

alias data="cd /data1/username/exp_data"
alias codes="cd /data1/username/codes"
alias ac_torch="conda activate pytorch1.5.0"

之后便可通过data和codes命令直接进入对应路径下

关于~/.bashrc文件，可以理解为每次登陆时自动执行一次其中的内容

直接在命令行执行export， alias等命令，有效期为本次登录期间。

可以将自定义命令写入另一个文件~/.my_cmd，然后在~/.bashrc最后加入

source ~/.my_cmd

alias命令不光可以在~/.bashrc即linux平台使用，在git, cmder等软件中均可以进行配置。

常用的命令比如ssh @， tensorboard都可以通过alias命令简化

PyTorch安装

首先使用nvidia-smi命令查看CUDA驱动版本，比如10.2

在PyTorch网页（链接）寻找对应的命令，建议使用conda命令安装，同时cudatoolkit版本与上述CUDA驱动版本保持一致

例如使用nvidia-smi命令后，显示版本为10.2, 则安装pytorch的命令为

conda install pytorch==1.5.0 torchvision==0.6.0 cudatoolkit=10.2 -c pytorch

注意要安装cudatoolkit包，不然可能导致torch.cuda.is_available()=False。

注意：PyTorch的安装不涉及到cuda和cudnn的配置，可以认为在cudatoolkit中实现了类似功能

Tensorflow安装

Tensorflow的安装的大体思路：配置cuda和cudnn, 然后利用export添加到路径，之后再安装tensorflow。

目前不太确定的是，你想要安装的cuda版本以及nvidia-smi显示的cuda版本以及系统/usr/local下的cuda版本之间的关系，以及是否会冲突等。

首先检查本机是否已经配置好cuda和cudnn：

  1. 检查cuda安装情况：执行nvcc -V命令，如果显示命令不存在，进行第2步；如果显示版本信息，进行第3步。
  2. cuda安装的确认：进入/usr/local/cuda/bin目录下，执行nvcc -V。如果显示了版本信息，说明cuda安装了，但是没有加入系统路径，之后如果在第3步确认cudnn也确实安装好了，可以将cuda加入系统路径，进入第3步。如果/usr/local/cuda*路径不存在，说明没有安装cuda。cuda都没有配置的话，cudnn基本也没有配置，非正常退出。
  3. 检查cudnn安装情况：在cuda目录下，执行cat /path/to/cuda/include/cudnn.h | grep CUDNNMAJOR -A5如果显示出CUDNN_MAJOR，CUDNN_MA等信息，则证明安装成功。说明本机已经配置好cuda和cudnn，正常退出。

cuda和cudnn环境准备好之后，即可安装tensorflow

pip install tensorflow-gpu==***

可以通过

import tensorflow as tf;tf.test.is_gpu_available()

检查，正确的输出结果为：

可以从log中看到GPU信息

cuda和cudnn的安装：

首先安装cuda。cuda仓库网址：CUDA Toolkit Archive

  1. 给文件运行权限 chmod + x .run, 然后运行./.run，依次选择accept—no—yes，然后将cuda安装到你的目录下（建议指定自己的目录）

  2. 下载cudnn。cudnn仓库网址：cuDNN Archive ​ 需要注册NVIDIA账号，根据名称，找对应刚才cuda版本的cudnn安装包下载解压。tar -xzvf cudnn-**-linux-x64-v**.tgz，win10下下载cudnn时后缀可能被修改，重命名为**.tgz即可。

  3. 进行部分文件的拷贝，从cudnn解压后的cuda文件夹，复制到之前的cuda文件夹中：

cp cuda/include/cudnn.h /path/to/your_cuda/include/
cp cuda/lib64/libcudnn* /path/to/your_cuda/lib64
chmod a+r /path/to/your_cuda/include/cudnn.h /path/to/your_cuda/lib64/libcudnn*
  1. 查看cuda和cudnn安装状态：分别为nvcc -V和

cat /path/to/your_cuda/include/cudnn.h | grep CUDNN_MAJOR -A5

如果输出正常版本号，则证明cuda和cudnn安装成功。

如果cuda安装成功后，即nvcc -V显示正确版本号，但是cudnn却失败了，可以尝试选用稍旧版本的cudnn而非最新版

tensorflow截止到1.15版本，都不支持cuda-10.2，为了安装tensorflow，需要手动配置cuda-10.0，并修改~/.bashrc文件中的PATH和LD_LIBRARY_PATH变量。

Linux常用命令

tmux：将程序放在后台执行，即使推出ssh登录，只要服务器没有关机，程序就不会终端。

创建新的窗口 tmux new -s your_name

暂时退出当前窗口 ctrl+b+d

结束终止当前窗口 exit

查看已创建的窗口 tmux ls

进入之前创建的窗口 tmux attach -t your_name或tmux a -t your_name

nohup：也是将程序放在后台执行，在没有tmux的情况下，使用nohup是一个退而求其次的选择

执行任务并放到后台 nohup bash yourscript.sh > your_log.log 2>&1 &

之后只能通过查看log文件，来了解程序运行输出情况。不能像tmux那样进入窗口查看。

top，htop，ps -aux：这三个都是用来查看本机运行的任务的，按需取用。htop和ps -aux可以比较方便的定位到具体的任务对应的PID进行，方便进行Kill

scp：用来进行数据传输。

服务器下载文件到本地：scp username@server_ip_address:文件在服务器的路径 文件在本地的保存路径

本机文件传递给服务器： scp 文件在本地的保存路径 username@server_ip_address:文件在服务器的路径

连接服务器需要指定端口时，加入-P （大写）参数

nvidia-smi，gpustat：这两个命令都可以用来查看当前机器的显卡占用情况，gpustat需要使用pip进行安装，界面简洁好看些。

tensorboard：用于查看训练过程中的结果

基本设置为：tensorboard --logdir path_to_log --port your_port

之后用浏览器打开 http://localhost:your_port即可，默认端口6006

打开服务器上的log文件

连接服务器：ssh -L 本地端口:127.0.0.1:服务器端口 username@server_ip_address

在服务器上执行：tensorboard --logdir path_to_log --port 服务器端口

在本地浏览器打开：http://localhost:本地端口

tensorboard的使用，需要pip安装tensorboard库，如果安装完之后还是没有，需要寻找对应的文件

alias tensorboard='python /home/username/anaconda3/envs/your_env_name/lib/python3.6/site-packages/tensorboard/main.py'

PyTorch笔记

  1. 关于device
  1. DataParallel会自动将输入数据映射到对应的模型device上，验证代码：
def verify_device():
    class net(nn.Module):
        def __init__(self):
            super(net, self).__init__()
        
        def forward(self, inputs):
            print("data's device in model: ", inputs.device)

    my_net = net()
    my_net = nn.DataParallel(my_net.cuda())  # my_net = my_net.cuda()
    inputs = torch.ones((16, 3, 256, 256))
    print("data's device out model: ", inputs.device)
    my_net(inputs)

利用DataParallel 只将模型放在GPU上，而没有将输入放在GPU上，输出结果如下：

data's device out model:  cpu
data's device in model:  cuda:0
data's device in model:  cuda:1
data's device in model:  cuda:2
data's device in model:  cuda:3
data's device in model:  cuda:5
data's device in model:  cuda:6
data's device in model:  cuda:7
data's device in model:  cuda:4

如果使用注释中的my_net = my_net.cuda()，则得到

data's device out model:  cpu
data's device in model:  cpu

BUG记录

  1. PyTorch的dataloader一旦执行

for iter, batch in enumerate(dataloader):

会卡住很久，进入__getitem__进行debug发现一直在读取图片，虽然我使用的并不是opencv库进行数据读取

解决办法：在主程序中加入cv2.setNumThreads(0)

谷歌搜索关键字：pytorch dataloader stucks

  1. Tensorflow多次load模型(为了进行多次测试)时，可能会出现

ValueError: Variable *** already exists, disallowed. Did you mean to set reuse=True or reuse=tf.AUTO_REUSE in VarScope?

这种情况，通常是需要在代码中加入，每次load模型前清理一次计算图tf.reset_default_graph()

  1. 基于PyTorch的代码，训练效果还不错，测试效果很糟糕，排除过拟合的原因以及测试代码的问题。最后发现是该代码在测试时需要指定

model.train()

  1. 程序报错：cudnn_status_not_initialized

可能是因为当前显卡正在被其他任务占用

  1. 安装neural_renderer_pytorch

RuntimeError: Error compiling objects for extension

解决办法：将pytorch的版本从1.5降到1.2

  1. 多卡GPU训练，保存的也是多卡模型，却需要单卡测试

Unexpected key(s) in state_dict: "module.*"

解决办法：

model.load_state_dict({k.replace('module.', ''): v for k, v in torch.load(model_state_path).items()})

  1. 模型在GPU上训练，需要在CPU上测试，跑inference

解决办法：

ckpt = torch.load(model_path, map_location=torch.device("cpu"))

  1. 程序报错: ModuleNotFoundError: No module named ‘network’

原因：保存模型时使用的是torch.save(model) 而不是torch.save(model.state_dict)

解决办法：在执行模型保存命令的py文件路径下，再次读取模型，并使用torch.save(model.state_dict)来保存模型
